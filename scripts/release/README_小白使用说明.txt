tts-video Windows x64 一键包使用说明
====================================

一、先解压
----------
请先把 zip 完整解压出来，不要直接在压缩包里双击运行。

建议放到英文路径，例如：
  D:\tts-video
  C:\AI\tts-video

不建议放到：
  中文路径
  OneDrive / 网盘同步目录
  Program Files 等权限受限目录
  路径里包含特殊符号的目录


二、启动
--------
双击 start.bat。

启动窗口会显示 IndexTTS API 和 WebUI 的合并日志。第一次启动时，IndexTTS 模型加载可能比较慢，请耐心等待。

启动成功后，浏览器会自动打开：
  http://127.0.0.1:8000


三、生成视频
------------
在网页中：

1. 上传角色图片。
2. 上传参考音频。
3. 输入要生成语音和字幕的文案。
4. 选择视频比例、背景和字幕样式。
5. 点击“生成视频”。
6. 生成完成后可在网页预览和下载。

生成结果保存位置：
  data\outputs\{task_id}\final.mp4


四、只使用 TTS 工具
------------------
如果只想生成语音，不需要生成视频，可以双击：
  start_tts.bat

首次使用默认启动 IndexTTS，并打开：
  http://127.0.0.1:9000

页面顶部可以在 IndexTTS 和 MiMoTTS 之间切换。切换时会停止当前 TTS 服务，再启动新的 TTS 工具。

IndexTTS 是本地模型推理。MiMoTTS 是小米 MiMo 云 API 音色克隆，会打开：
  http://127.0.0.1:9021

如果 MiMoTTS 页面提示填写 API Key，请输入你的 MiMo API Key。保存后会写入当前系统用户环境变量 MIMO_API_KEY，后续页面不会再显示这个 Key。

使用 MiMoTTS 时，文本和参考音频会发送到 MiMo API 服务。请只上传本人或已获授权的音频素材。


五、关闭
--------
推荐两种方式：

1. 在启动窗口按 Q。
2. 或双击 stop.bat。

stop.bat 只会停止本项目 runtime 目录里 PID 文件记录的进程，不会乱杀其他占用 8000 / 9000 端口的程序。


六、常见问题
------------

1. 提示 8000 或 9000 端口被占用
   说明你的电脑上已有其他程序占用了端口。请手动关闭那个程序后再启动。

2. Windows Defender / 杀毒软件误报
   本包内包含 Python、FFmpeg、PyTorch、CUDA 运行库和模型文件，体积较大，杀毒软件可能会扫描较久或误报。请确认下载来源可信。

3. 显存不足 / CUDA OOM
   关闭其他占用显卡的软件，缩短文案，换更大显存显卡，或等待后续版本提供更细的推理配置。

4. 首次加载很慢
   正常。IndexTTS 模型第一次加载需要初始化大量模型权重，后续请求通常会快一些。

5. 路径包含中文或特殊字符导致问题
   请把整个文件夹移动到英文路径，例如 D:\tts-video。

6. 不要移动内部目录
   请不要单独移动 runtime、index-tts、configs、modules、external、static 等目录。它们之间使用相对路径配合运行。
